# Chrome Web Store 提出ガイド — PSA 返却予定日ヘルパー

Chrome Web Store へアップロードする際の手順とチェックリスト。ローカルの動作確認が済んだ後に本ドキュメントに沿って進める。

## 1. 事前準備

- [ ] Chrome Web Store デベロッパー登録（$5 の登録料、初回のみ）
- [ ] PRIVACY.md をホスティング先 (GitHub Pages / 個人サイト / Notion 公開ページ等) に公開し、公開 URL を控える
- [ ] 掲載用スクリーンショット (1280×800 または 640×400) を 1〜5 枚用意
- [ ] 128×128 のストア用アイコン (icons/icon-128.png をそのまま利用可、必要に応じてデザイン差し替え)
- [ ] `manifest.json` の `version` を確認 (再アップロード時は必ずインクリメント)

## 2. パッケージ生成

```bash
node scripts/build-zip.mjs
```

`dist/psa-return-eta-vX.Y.Z.zip` が生成される。以下だけを含む:

- `manifest.json`
- `icons/`
- `src/`

`fixtures/`, `SPEC.md`, `TASKS.md`, `CLAUDE.md`, `README.md`, `PRIVACY.md`, `STORE.md`, `.claude/`, `.git/` などは自動的に含まれない。

## 3. Chrome Web Store デベロッパーダッシュボードでの入力項目

### 商品情報
- **カテゴリ**: 「生産性 (Productivity)」推奨
- **言語**: 日本語 (ja) を主言語に、必要なら英語も追加
- **概要 (短)**: 「PSA ご注文一覧に返却予定日を追記表示するヘルパー拡張」
- **説明 (長)**: 主要機能 3 点 + 免責事項 (PSA 非公式)

### プライバシー
- **単一目的の説明**: 「PSA ご注文一覧ページに、TAT と PSA 到着日から順算した返却予定日を表示する」
- **権限の理由**:
  - `storage`: ユーザーが編集した TAT・取得済みマスタ・祝日情報のローカル保存
  - `alarms`: TAT / 祝日情報の日次・週次バックグラウンド更新
  - `www.psacard.com/ja-JP/*`: 対象ページの読み取り、`/services` からの TAT マスタ取得
  - `www8.cao.go.jp/*`: 内閣府の祝日 CSV 取得
- **リモートコード**: 「使用しない」
- **データの収集・使用**: いずれもオフ（外部送信ゼロ）
- **プライバシーポリシー URL**: PRIVACY.md を公開した URL を入力

### 配布
- **公開範囲**: 公開 / 限定公開 / 非公開 のいずれか。テスト時は「限定公開」推奨。
- **対象地域**: 日本のみ / 全世界 のどちらか（PSA 非公式のため公式表記に注意）

## 4. 公開前チェックリスト

- [ ] 実際にビルドした zip を「パッケージ化されていない拡張機能」ではなく、`chrome://extensions` の「パッケージ化された拡張機能を読み込む」相当（drag & drop）で読み込み、動作を再確認
- [ ] `console.warn('[psa-ext] ...')` 以外のエラーが出ないこと
- [ ] PSA ご注文一覧ページ以外では一切の DOM 変更が起きないこと
- [ ] 拡張アイコンがツールバー・拡張機能一覧に表示されること
- [ ] PRIVACY.md の連絡先を実在のものに置き換えたこと

## 5. 商標・法的注意

- 「PSA」は Collectors Universe, Inc. の商標。本拡張は非公式であり、名称・説明に「非公式 (unofficial)」を明記する。
- 掲載スクリーンショットに PSA のロゴを含める場合は自作アイコンで代替する方が安全。
